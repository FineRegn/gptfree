## 1. 技术目标

1. **全量数据覆盖**：5 个 Tab 的数据均来自后端独立接口，前端不直接查库
2. **埋点最小侵入**：在已有的 `get_verification_result` 函数（`external_api.py`）最外层包裹埋点，不修改内部提取逻辑
3. **写入失败不阻塞主流程**：日志写入失败不影响验证码提取结果返回
4. **懒加载策略**：前端只在 Tab 首次激活时请求数据，减少进入页面时的请求开销
5. **DB 迁移幂等**：v23 迁移使用 `CREATE TABLE IF NOT EXISTS`，重复执行安全

## 4. 验证码提取埋点设计

### 4.1 修改文件

`outlook_web/services/external_api.py`

### 4.2 埋点位置：`get_verification_result` 最外层

`get_verification_result` 是**普通账号验证码提取的统一埋点入口**。主应用页面里的 `/api/emails/<email>/extract-verification` 已改为复用这里的逻辑；临时邮箱 `/api/temp-emails/<email>/extract-verification` 仍保留独立 service，但已复用同一张 `verification_extract_logs` 表。补充一点：主应用 Web 提取入口虽然接入了 shared logging，但仍保留旧的内部错误语义与测试 patch 兼容点（如 `EMAIL_NOT_FOUND` / `IMAP_AUTH_FAILED`），不与 external API 的 `UPSTREAM_READ_FAILED` 契约完全等同。

**改动方式**（在函数末尾统一包裹）：

```python
def get_verification_result(
    *,
    email_addr: str,
    ...
) -> Dict[str, Any]:
    # ---- 埋点：记录开始时间 ----
    _extract_started_at = time.time()
    _extract_account_id: Optional[int] = None
    _extract_channel: str = "unknown"
    _extract_used_ai: bool = False
    _extract_error_code: Optional[str] = None

    try:
        result = _get_verification_result_inner(...)  # 重命名原函数体为 _inner
        # 从 result 中读取埋点字段
        _extract_channel = result.get("_log_channel") or "unknown"
        _extract_used_ai = bool(result.get("_log_used_ai"))
        return result
    except Exception as exc:
        _extract_error_code = _classify_extract_error(exc)
        raise
    finally:
        _write_extract_log(
            account_id=_extract_account_id,
            channel=_extract_channel,
            started_at=_extract_started_at,
            finished_at=time.time(),
            result=result if not _extract_error_code else None,
            used_ai=_extract_used_ai,
            error_code=_extract_error_code,
        )
```

> **注意**：实际实现保留了内部辅助函数 `_write_extract_log()`，但采用同步写库 + `try/except` 吞异常；不会阻断主流程。
>
> **再次核对后的现状**：自动化测试验证的是共享入口的埋点行为；当前普通账号前端 UI 已并入这条链路，临时邮箱则通过负数 `temp_emails.id` 哨兵值写入同一张日志表。

### 4.3 `_write_extract_log` 实现

新增私有函数，在 `external_api.py` 底部（或提取到独立 helper 模块）：

```python
def _write_extract_log(
    *,
    account_id: Optional[int],
    channel: str,
    started_at: float,
    finished_at: float,
    result: Optional[Dict[str, Any]],
    used_ai: bool,
    error_code: Optional[str],
    trace_id: Optional[str] = None,
) -> None:
    """写入验证码提取日志，失败时吞异常，不影响主流程。"""
    try:
        conn = get_db()
        duration_ms = int(round((finished_at - started_at) * 1000))

        if result:
            code = result.get("verification_code") or result.get("formatted")
            link = result.get("verification_link")
            result_type = "code" if code else ("link" if link else "none")
            code_found = code or link
        else:
            result_type = "none"
            code_found = None

        conn.execute(
            """
            INSERT INTO verification_extract_logs
              (account_id, channel, started_at, finished_at, duration_ms,
               result_type, code_found, used_ai, error_code, trace_id)
            VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
            """,
            (account_id, channel, started_at, finished_at, duration_ms,
             result_type, code_found, int(used_ai), error_code, trace_id),
        )
        conn.commit()
    except Exception:
        pass  # 吞异常，不影响主流程
```

### 4.4 channel 字段来源

为让埋点记录到实际使用的通道（而非路由计划中的通道），需要在提取结果中透传 `_log_channel` 字段：

- `extract_verification_for_outlook`（`verification_channel_routing.py`）在成功提取后，在返回值中新增 `"_log_channel": channel`（仅用于埋点，非对外字段）
- 通用 IMAP 和临时邮箱路径类似处理

```javascript
// 模块级状态
const _cache = {};     // { tabId: data }
let _activeTab = null;

function initOverview() {
    // 绑定 Tab 切换点击
    document.querySelectorAll('.ov-tab').forEach(btn => {
        btn.addEventListener('click', () => switchOverviewTab(btn.dataset.tab));
    });
    // 绑定刷新按钮
    const refreshBtn = document.getElementById('ov-refresh-btn');
    if (refreshBtn) refreshBtn.addEventListener('click', refreshOverview);
    // 默认加载总览
    switchOverviewTab('summary');
}

function switchOverviewTab(tabId) {
    // 切换 active 状态
    document.querySelectorAll('.ov-tab').forEach(b => b.classList.toggle('active', b.dataset.tab === tabId));
    document.querySelectorAll('.ov-tab-pane').forEach(p => p.classList.toggle('active', p.id === `tab-${tabId}`));
    _activeTab = tabId;
    // 懒加载
    if (!_cache[tabId]) {
        _loadTab(tabId);
    }
}

function refreshOverview() {
    Object.keys(_cache).forEach(k => delete _cache[k]);
    if (_activeTab) _loadTab(_activeTab);
    _updateRefreshTime();
}

function _loadTab(tabId) {
    const handlers = {
        summary:          loadOverviewSummary,
        'verification':   loadVerificationStats,
        'external-api':   loadExternalApiStats,
        pool:             loadPoolStats,
        activity:         loadActivityStats,
    };
    const fn = handlers[tabId];
    if (fn) fn();
}

async function loadOverviewSummary() {
    const data = await _fetch('/api/overview/summary');
    if (data) { _cache.summary = data; _renderSummary(data); }
}
// ... 其他 4 个 loadXxx 函数

function _renderSummary(data) { /* 渲染总览 Tab */ }
function _renderVerification(data) { /* 渲染验证码 Tab */ }
function _renderExternalApi(data) { /* 渲染对外 API Tab，含纯 CSS 柱图 */ }
function _renderPool(data) { /* 渲染邮箱池 Tab */ }
function _renderActivity(data) { /* 渲染系统活动 Tab，含时间线 */ }

async function _fetch(url) {
    try {
        const resp = await fetch(url);
        const json = await resp.json();
        return resp.ok ? json : null;
    } catch { return null; }
}

function _updateRefreshTime() {
    const el = document.getElementById('ov-last-refresh');
    if (el) el.textContent = new Date().toLocaleTimeString('zh-CN');
```

当前实际前端渲染还增加了两类 UI helper：

1. `renderDataCard(options)`：统一输出 overview 玻璃数据卡片。
2. `renderHoverNote(text)`：统一输出卡片 hover 浮层说明。
3. `notifyOverviewDataChanged(tabIds, reason)`：在提取等关键动作成功后，主动让 overview 指定缓存失效。

本轮视觉收敛中，overview 没有切到冷白蓝灰路线，而是继续贴合项目现有暖色品牌基底：通过降低饱和度与透明度，让新大盘在玻璃质感下仍能与主站主题保持一致。

此外，前端数据新鲜度策略已从“纯命中缓存”调整为：

1. 进入 dashboard 时，先显示已有缓存，再对当前 Tab 强制发起一次真实后端请求。
2. 在验证码提取成功后，通过 `overview-data-changed` 事件失效 `summary` / `verification` / `activity` 缓存。
3. 若大盘当前可见，则立刻重拉当前 Tab，保证页面尽快反映数据库最新状态。

补充说明（按当前真实运行边界）：

1. 这里的 `overview-data-changed` 只存在于主应用页面前端上下文中，当前由 `static/js/features/groups.js` 在站内提取成功后触发。
2. 这个事件负责让 overview 前端缓存失效并重新 fetch overview API；主应用前端 UI 的普通账号与临时邮箱提取现在都已经同步写入 `verification_extract_logs`。

### 10.1 向后兼容

- 旧 `page-dashboard` HTML 被完整替换，但 `.stat-grid` / `.dashboard-grid` CSS 类**不删除**

- `_write_extract_log` 内部 `try/except` 吞掉所有异常

### 10.3 DB 迁移幂等性

- 所有迁移 SQL 使用 `CREATE TABLE IF NOT EXISTS` + `CREATE INDEX IF NOT EXISTS`
- 重复执行不会报错
