## 1. 文档目标

本 TDD 只关注「数据概览大盘」的测试设计，不重复展开业务背景与技术方案。

核心要证明以下六件事：

1. **DB v23 迁移正确且幂等**：`verification_extract_logs` 表与 4 个索引在新库和旧库上均能成功创建
2. **埋点写入准确**：`_write_extract_log` 写入正确字段，异常时不影响主流程
3. **`_log_channel` 透传正确**：`verification_channel_routing.py` 的通道标识能正确传递到日志
4. **5 个 Overview Repository 查询正确**：有数据时统计正确，无数据时返回合法空结构
5. **5 个 Overview API 接口契约稳定**：鉴权正常，响应结构与代码实际契约对齐
6. **现有主流程不回退**：验证码提取、外部 API、邮箱池等核心链路不受本次改动影响

## 3. 测试原则

1. **先测数据层，再测接口层**：Repository 正确是 API 正确的前提，不能跳过 C 层直接用 D 层集成覆盖
2. **边界即重点**：空表 / 空数据 / 无最近记录 时不能 500，要验证「合法空值返回」
3. **埋点失败安全性必须有用例**：`_write_extract_log` 的异常吞掉是设计意图，必须有测试显式覆盖
4. **幂等迁移必须独立验证**：DB 迁移的幂等性不能靠 Repository 测试间接证明
5. **不引入新测试基础设施**：复用现有 `tests/_import_app.py`、`tests.test_utils`（若存在）中的工具函数
